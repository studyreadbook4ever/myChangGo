# 잠깐 맡김소 사용 가이드

잠깐 맡김소는 지금 처리할 수는 없지만 잊고 싶지도 않은 생각을 잠시 저장해 두는 앱입니다. `생각 내용`과 다시 볼 시간을 적어 두면, 고른 시간이 되었을 때 해당 생각을 `지금 볼 생각`으로 옮겨 줍니다.

계정이나 서버 없이 현재 브라우저에만 저장되므로, 짧은 할 일이나 집중을 방해하는 생각을 가볍게 내려놓을 때 잘 맞습니다.

> [!IMPORTANT]
> 잠깐 맡김소는 시스템 알림이나 푸시 알림을 보내지 않습니다. 고른 시간은 앱을 열어 두었거나 다시 열었을 때 `지금 볼 생각`으로 보여 주기 위한 기준입니다.

## 빠른 시작

1. `새 생각`의 `생각 내용` 입력란에 한 줄을 적습니다.
2. `10분 뒤`, `1시간 뒤`, `오늘 저녁`/`내일 저녁`, `내일 아침`, `날짜 직접 선택` 중 다시 볼 시간을 고릅니다. 기본값은 `1시간 뒤`입니다.
3. `생각 저장하기`를 누릅니다. 키보드에서는 `Ctrl`/`⌘` + `Enter`로도 저장할 수 있습니다.
4. 저장 직후에는 `나중에 볼 생각`에 나타납니다.
5. 고른 시간이 되면 `지금 볼 생각`으로 이동합니다.
6. 처리했다면 `완료`, 아직 아니라면 `나중에 보기`를 선택합니다.

처음 저장된 생각이 하나도 없으면 `저장한 생각` 아래에 첫 사용 안내 카드가 하나 나타납니다. `예시 문장 넣어보기`를 누르면 예시가 `생각 내용` 입력란에 채워집니다. 바로 저장되지는 않으므로 내용을 고친 뒤 시간과 함께 직접 저장하면 됩니다.

## 처음부터 끝까지: 현실적인 사용 예

오후 2시 20분, 보고서에 집중하던 중 “퇴근 전에 민지에게 계약서 최종본을 보내야 한다”는 생각이 떠올랐다고 해 보겠습니다.

### 1. 새 생각 저장하기

입력란에 다음 내용을 적습니다.

> 민지에게 계약서 최종본 보내기

오후 5시 30분에 다시 보고 싶으므로 `날짜 직접 선택`을 누르고 오늘 날짜의 `17:30`을 선택한 다음 `생각 저장하기`를 누릅니다.

저장한 내용이 `나중에 볼 생각`에 생기고 다음 정보가 함께 보입니다.

- `나중에 볼 생각` 상태
- 생각 내용
- 다시 볼 상대 시간과 정확한 날짜·시간
- 처음 저장한 날짜·시간

이제 보고서 작업으로 돌아가도 됩니다.

### 2. 필요할 때 찾기

오후 4시쯤 내용을 확인하고 싶다면 검색란에 `계약서`를 입력합니다. `생각 내용`에 그 단어가 들어간 항목만 상태별로 남습니다.

검색 결과를 보고 바로 처리할 수 있다면 `완료`를 눌러도 됩니다. 아직 아니라면 검색어를 지우고 그대로 둡니다.

### 3. 고른 시간에 다시 보기

오후 5시 30분이 되면 저장한 내용이 `나중에 볼 생각`에서 `지금 볼 생각`으로 이동합니다. 앱이 열려 있다면 최대 약 30초 안에 화면이 갱신되고 안내 문구가 나타납니다. 앱이 닫혀 있었다면 다음에 열 때 바로 현재 시간에 맞춰 분류됩니다.

마침 통화 중이라 한 시간 뒤에 처리하고 싶다면 다음과 같이 선택합니다.

1. `나중에 보기`를 엽니다.
2. `1시간 뒤`를 누릅니다.
3. 생각이 다시 `나중에 볼 생각`으로 이동했는지 확인합니다.

### 4. 완료하고, 필요하면 되돌리기

오후 6시 30분에 파일을 보낸 뒤 `완료`를 누릅니다. 해당 내용은 접힌 `완료한 생각` 영역으로 이동합니다.

버튼을 잘못 눌렀다면 화면 아래 안내가 사라지기 전, 약 10초 안에 `이전 작업 되돌리기`를 누릅니다. 제대로 완료한 것이 맞다면 그대로 두면 됩니다.

나중에 같은 일을 다시 확인해야 한다면 `완료한 생각`을 열고 `다시 열기`를 누릅니다. 해당 생각은 현재 시각을 기준으로 **10분 뒤** 다시 보도록 `나중에 볼 생각`으로 이동합니다.

### 5. 하루가 끝나면 백업하기

중요한 내용이 있다면 상단의 `데이터` 버튼을 열고 `내보내기`를 누릅니다. 모든 상태의 저장한 생각이 날짜가 붙은 JSON 파일로 내려받아집니다.

예:

```text
jamkkan-matgimso-2026-07-24.json
```

이 파일을 개인 백업 폴더에 보관하면 브라우저를 바꾸거나 사이트 데이터가 지워졌을 때 다시 가져올 수 있습니다.

## 화면 구성

| 영역 | 하는 일 |
| --- | --- |
| 새 생각 | `생각 내용`을 최대 280자까지 적고 다시 볼 시간을 정한 뒤 `생각 저장하기`를 누릅니다. |
| 저장한 생각 요약 | 전체 `지금 볼 생각`, `나중에 볼 생각`, `완료한 생각` 개수를 보여 줍니다. 저장한 내용이 있을 때 표시됩니다. |
| 검색 | 생각 내용에 포함된 단어로 세 상태를 모두 검색합니다. |
| 지금 볼 생각 | 고른 시각이 현재와 같거나 지난 생각을 보여 줍니다. |
| 나중에 볼 생각 | 아직 고른 시각이 오지 않은 생각을 보여 줍니다. |
| 완료한 생각 | `완료`한 생각을 접을 수 있는 목록으로 보여 줍니다. |
| 사용법 | 세 단계 사용 흐름과 상태 뜻을 간단히 보여 주고 이 문서를 엽니다. |
| 데이터 | 설치 지원 시 `앱 설치`, JSON 내보내기·가져오기, 전체 삭제를 제공합니다. |

상단의 `이 브라우저에 저장` 배지는 브라우저 저장 공간을 정상적으로 사용하고 있다는 뜻입니다. 저장이 차단되면 배지가 `이 탭에서만 유지`로 바뀌고, 작은 화면을 포함해 `저장 차단됨` 경고가 계속 표시됩니다. 이때는 [저장이 되지 않을 때](#저장이-되지-않을-때)를 확인하세요.

## 새 생각 저장하기

### 내용 적기

- 공백만 있는 내용은 저장할 수 없습니다.
- 앞뒤 공백은 저장할 때 제거됩니다.
- 한 생각에는 최대 280자까지 적을 수 있습니다.
- 입력란 아래에서 현재 글자 수를 확인할 수 있습니다.
- 직접 수정 기능은 없습니다. 저장한 문구를 바꾸려면 기존 생각을 삭제한 뒤 새로 저장해야 합니다.

짧고 나중에 행동으로 옮기기 쉬운 문장이 좋습니다.

| 막연한 표현 | 더 알아보기 쉬운 표현 |
| --- | --- |
| 발표 준비 | 발표 첫 장에 핵심 메시지 한 줄 쓰기 |
| 병원 | 점심시간에 치과 예약 전화하기 |
| 엄마 | 저녁에 엄마에게 주말 일정 물어보기 |

### 다시 볼 시간 고르기

모든 시간은 현재 기기와 브라우저가 사용하는 현지 시각을 기준으로 계산됩니다.

| 선택 | 실제로 정해지는 시각 |
| --- | --- |
| `10분 뒤` | 저장한 순간으로부터 정확히 10분 뒤 |
| `1시간 뒤` | 저장한 순간으로부터 정확히 1시간 뒤 |
| `오늘 저녁` 또는 `내일 저녁` | 오후 8시 전에는 `오늘 저녁`, 오후 8시가 지나면 `내일 저녁`으로 표시. 실제 시간은 각각 오늘 또는 다음 날 오후 8시 |
| `내일 아침` | 다음 달력 날짜의 오전 9시 |
| `날짜 직접 선택` | 날짜와 시간을 직접 선택. 현재보다 뒤여야 함 |

`날짜 직접 선택`을 처음 열면 약 2시간 뒤가 제안됩니다. 과거 또는 현재 시각을 고르면 저장되지 않으며 `지금보다 뒤의 날짜와 시간을 골라 주세요`라는 안내가 표시됩니다.

> [!NOTE]
> `오늘 저녁`/`내일 저녁`과 `내일 아침`은 고정된 달력 시각입니다. “24시간 뒤”처럼 단순히 시간을 더하는 방식이 아니므로, 자정이나 일광 절약 시간 변경이 있는 지역에서도 기기의 현지 날짜를 따릅니다.

### 저장하기

다음 중 한 가지 방법을 사용합니다.

- 화면의 `생각 저장하기` 누르기
- Windows/Linux에서 `Ctrl` + `Enter`
- macOS에서 `⌘` + `Enter`

저장되면 입력란과 시간 선택이 초기화되고 기본값은 다시 `1시간 뒤`가 됩니다. 저장 직후에는 화면 아래의 `이전 작업 되돌리기`로 방금 만든 생각을 취소할 수 있습니다.

## 세 가지 상태 이해하기

| 상태 | 의미 | 정렬 방식 | 가능한 동작 |
| --- | --- | --- | --- |
| `나중에 볼 생각` | 다시 볼 시각이 아직 오지 않음 | 가장 빨리 볼 생각부터 | 완료, 지금 보기, 삭제 |
| `지금 볼 생각` | 다시 볼 시각이 현재와 같거나 이미 지남 | 가장 오래 기다린 생각부터 | 완료, 나중에 보기, 삭제 |
| `완료한 생각` | 사용자가 처리를 마쳤다고 표시함 | 가장 최근에 완료한 생각부터 | 다시 열기, 삭제 |

`지금 볼 생각`은 실패나 지연을 뜻하지 않습니다. 단지 “이 생각을 다시 볼 때가 되었다”는 분류입니다. 시간이 지나도 자동으로 삭제되거나 완료되지 않습니다.

상단 요약의 세 숫자는 검색 여부와 관계없이 저장한 생각 전체의 개수입니다. 검색 중 각 목록 옆 숫자는 현재 검색 결과에 보이는 개수입니다.

## 저장한 생각의 모든 동작

### `나중에 볼 생각`에서

#### 완료

고른 시간 전이지만 이미 처리한 경우 사용합니다. 해당 내용이 `완료한 생각`으로 이동하고 완료한 시각이 기록됩니다.

#### 지금 보기

기다리지 않고 바로 확인하고 싶을 때 사용합니다. 다시 볼 시각이 현재로 바뀌며 해당 내용이 곧바로 `지금 볼 생각`으로 이동합니다.

#### 삭제

해당 생각을 지웁니다. 화면 아래 안내가 보이는 약 10초 동안에는 되돌릴 수 있습니다.

### `지금 볼 생각`에서

#### 완료

생각을 처리했다고 표시합니다. 해당 내용이 `완료한 생각`으로 이동합니다.

#### 나중에 보기

아직 처리할 때가 아니라면 다음 중 하나를 고릅니다.

| 선택 | 새로 볼 시간 |
| --- | --- |
| `10분 뒤` | 누른 시각으로부터 10분 뒤 |
| `1시간 뒤` | 누른 시각으로부터 1시간 뒤 |
| `내일 이맘때` | 누른 시각으로부터 24시간 뒤 |

미루고 나면 상태는 다시 `나중에 볼 생각`이 됩니다.

#### 삭제

해당 생각을 지웁니다. 약 10초 안에는 되돌릴 수 있습니다.

### `완료한 생각`에서

#### 다시 열기

완료한 생각을 다시 확인해야 할 때 사용합니다. 상태가 `나중에 볼 생각`으로 바뀌고, 다시 볼 시간은 누른 순간으로부터 10분 뒤로 정해집니다.

#### 삭제

완료 기록을 포함해 해당 생각을 지웁니다. 약 10초 안에는 되돌릴 수 있습니다.

### 전체 삭제

데이터 관리에서 `전체 삭제`를 누르면 모든 상태의 저장한 생각을 한꺼번에 지울 수 있습니다. 확인 창에서 `모두 지우기`를 눌러야 실제로 삭제됩니다.

> [!WARNING]
> 전체 삭제는 화면 아래의 되돌리기 대상이 아닙니다. 필요한 데이터가 있다면 먼저 `내보내기`로 백업하세요.

## 검색하기

1. 검색란에 기억나는 단어 일부를 입력합니다.
2. `지금 볼 생각`, `나중에 볼 생각`, `완료한 생각` 전체에서 `생각 내용`이 일치하는 항목이 표시됩니다.
3. 결과가 없으면 `검색 지우기`를 눌러 저장한 생각 전체로 돌아갑니다.

검색은 다음 원칙으로 동작합니다.

- 생각 **본문**에서 부분 일치로 찾습니다.
- 영문 대소문자는 구분하지 않습니다.
- 상태 이름, 날짜와 시간은 검색 대상이 아닙니다.
- 검색 중에도 보이는 생각의 버튼을 그대로 사용할 수 있습니다.
- 완료한 내용이 검색되면 `완료한 생각` 영역이 자동으로 열립니다.

예를 들어 `민지에게 계약서 최종본 보내기`는 `민지`, `계약서`, `최종본` 중 어느 것을 입력해도 찾을 수 있습니다.

키보드를 사용한다면 입력란이나 다른 편집 필드에 커서가 없을 때 `/`를 눌러 검색란으로 바로 이동할 수 있습니다.

## 마지막 작업 되돌리기

생각을 저장하거나 상태를 바꾸거나 삭제하면 화면 아래에 약 10초 동안 안내가 나타납니다. 이때 `이전 작업 되돌리기`를 누르면 바로 직전 상태로 돌아갑니다.

되돌릴 수 있는 대표 작업은 다음과 같습니다.

- 새 생각 저장
- `완료`
- `지금 보기`
- `나중에 보기`
- `다시 열기`
- 생각 하나 삭제
- 정상 상태에서 JSON 합치기 또는 교체

다음 작업은 되돌릴 수 없습니다.

- 데이터 관리의 `전체 삭제`
- 손상 데이터를 읽힌 항목만으로 복구
- 읽지 못한 원본 데이터를 비우고 새로 시작
- 손상 데이터 복구 중 백업 파일로 교체

> [!NOTE]
> 되돌리기는 한 번에 가장 최근 작업 하나만 기억합니다. 10초가 지나거나 그사이에 다른 작업을 하면 이전 되돌리기 기회는 사라집니다.

## 키보드 단축키

| 키 | 동작 | 조건 |
| --- | --- | --- |
| `N` | 새 생각 입력란으로 이동 | 대화상자가 닫혀 있고 텍스트를 편집 중이지 않을 때 |
| `/` | 검색란으로 이동하고 기존 검색어 선택 | 대화상자가 닫혀 있고 텍스트를 편집 중이지 않을 때 |
| `Ctrl`/`⌘` + `Enter` | 작성 중인 생각 저장하기 | 대화상자가 닫혀 있을 때 |
| `Esc` | 열린 사용법·데이터 관리·확인 창 닫기 | 대화상자가 열려 있을 때 |
| `Esc` | `생각 내용` 입력란에서 포커스만 해제 | 입력한 내용은 그대로 유지됨 |

한글 입력기의 조합 중 Enter 키는 저장 단축키로 잘못 처리되지 않습니다. 버튼과 입력 요소는 `Tab`과 `Shift` + `Tab`으로도 이동할 수 있습니다. `Esc`는 작성 취소나 내용 삭제가 아니므로, 다시 입력란으로 이동하면 이어서 작성할 수 있습니다.

## 백업과 가져오기

### JSON으로 내보내기

1. 상단의 `데이터` 버튼을 누릅니다.
2. `보관함 옮기기`에서 `내보내기`를 누릅니다.
3. 내려받은 `.json` 파일을 안전한 위치에 보관합니다.

내보내기 파일에는 `나중에 볼 생각`, `지금 볼 생각`, `완료한 생각`이 모두 포함됩니다. 파일은 암호화되지 않은 일반 텍스트이므로 민감한 내용이 있다면 보관 위치에 주의하세요.

내보낸 데이터는 대략 다음과 같은 구조입니다.

```json
{
  "schemaVersion": 1,
  "exportedAt": 1784887200000,
  "items": [
    {
      "id": "example-contract-001",
      "text": "민지에게 계약서 최종본 보내기",
      "createdAt": 1784870400000,
      "reviewAt": 1784881800000,
      "status": "done",
      "doneAt": 1784885400000
    },
    {
      "id": "example-plant-002",
      "text": "집에 가면 화분 흙이 마르지 않았는지 보기",
      "createdAt": 1784871000000,
      "reviewAt": 1784890800000,
      "status": "holding",
      "doneAt": null
    }
  ]
}
```

각 저장 항목의 핵심 값은 다음과 같습니다.

| 값 | 의미 |
| --- | --- |
| `id` | 저장한 생각을 구분하는 고유 값 |
| `text` | 사용자가 적은 생각 |
| `createdAt` | 저장한 시각 |
| `reviewAt` | 다시 볼 시각 |
| `status` | 내부 저장 상태. `holding` 또는 `done` |
| `doneAt` | 완료한 시각. 미완료이면 `null` |

화면의 `나중에 볼 생각`과 `지금 볼 생각`은 모두 내부적으로 `status: "holding"`입니다. 둘은 `reviewAt`이 현재보다 미래인지, 현재와 같거나 과거인지에 따라 나뉩니다. `완료한 생각`은 `status: "done"`이며 `doneAt`에 완료 시각이 들어갑니다.

시각 값은 밀리초 단위의 숫자로 저장됩니다. 직접 편집하면 파일을 읽지 못하거나 항목이 제외될 수 있으므로, 가능하면 앱에서 생성한 파일을 그대로 사용하세요.

### JSON 가져오기

1. 데이터 관리를 열고 `가져오기`를 누릅니다.
2. 잠깐 맡김소에서 내보낸 JSON 파일을 선택합니다.
3. 파일에서 읽을 수 있는 저장 항목 수를 확인합니다.
4. `기존 보관함에 합치기` 또는 `파일 내용으로 교체`를 고릅니다.

| 선택 | 결과 | 알맞은 상황 |
| --- | --- | --- |
| 기존 보관함에 합치기 | 현재 저장한 생각을 유지하고 파일의 항목을 추가 | 두 브라우저의 내용을 한곳에 모을 때 |
| 파일 내용으로 교체 | 현재 저장한 생각 전체를 파일 속 내용으로 바꿈 | 백업 시점으로 복원하거나 다른 브라우저로 그대로 옮길 때 |
| 취소 | 아무것도 바꾸지 않음 | 파일이나 선택을 다시 확인할 때 |

같은 `id`를 가진 항목은 중복으로 추가되지 않습니다. 파일에 손상되거나 중복된 항목이 섞여 있으면 읽을 수 있는 항목 수와 제외된 항목 수를 확인 창에서 알려 줍니다.

> [!CAUTION]
> `파일 내용으로 교체`는 현재 저장한 생각 전체를 바꿉니다. 정상 데이터에서 교체한 직후에는 약 10초간 되돌릴 수 있지만, 그 기회에 의존하지 말고 중요한 현재 데이터는 먼저 별도 파일로 내보내세요.

다음 파일은 가져오지 않습니다.

- JSON 문법이 아닌 파일
- 저장 항목 배열을 포함하지 않은 형식
- 유효한 저장 항목이 하나도 없는 파일
- 현재 앱보다 더 새로운 데이터 형식으로 내보낸 파일

## 오프라인으로 설치하고 사용하기

### 먼저 알아둘 점

오프라인 준비와 설치에는 서비스 워커가 사용됩니다. 따라서 다음 중 하나로 앱을 열어야 합니다.

- 배포된 HTTPS 주소
- `http://localhost`의 로컬 정적 서버

`file://`로 `index.html`을 직접 여는 방식에서는 서비스 워커가 동작하지 않습니다.

### 설치하기

1. 네트워크에 연결된 상태에서 앱을 한 번 완전히 엽니다.
2. 상단의 `데이터` 버튼을 엽니다.
3. `앱으로 설치하기` 영역에 `앱 설치`가 보이면 누르고 브라우저의 설치 안내를 따릅니다.
4. 이 영역이 보이지 않으면 브라우저 메뉴에서 `앱 설치`, `홈 화면에 추가` 또는 비슷한 항목을 찾습니다.
5. iPhone/iPad의 Safari에서는 공유 메뉴를 열고 `홈 화면에 추가`를 선택합니다.
6. 설치 뒤 홈 화면, 앱 목록 또는 바탕 화면의 `잠깐 맡김소` 아이콘으로 실행합니다.

앱 안의 설치 영역은 현재 브라우저가 설치를 지원하고 설치 조건이 갖춰졌을 때만 데이터 관리 대화상자 안에 표시됩니다. 브라우저와 운영체제에 따라 설치 기능을 제공하지 않거나 설치 버튼의 이름이 다를 수 있습니다. 설치하지 않아도 일반 브라우저 탭에서 같은 기능을 사용할 수 있습니다.

### 오프라인 사용 준비 확인

처음 방문한 직후 앱 화면이 완전히 열린 것을 확인하고 한 번 새로고침하거나 닫았다 다시 여는 것을 권합니다. 그다음 다음 순서로 확인할 수 있습니다.

1. 생각 하나를 테스트로 저장합니다.
2. 비행기 모드 또는 네트워크 끊기를 켭니다.
3. 설치한 앱이나 같은 주소를 다시 엽니다.
4. 화면이 열리고 테스트로 저장한 생각이 남아 있는지 확인합니다.

앱 화면 파일은 오프라인 캐시에, 저장한 생각은 브라우저의 로컬 저장소에 각각 보관됩니다. 온라인일 때는 최신 파일을 우선 확인하고, 연결할 수 없을 때 저장해 둔 화면 파일을 사용합니다.

> [!IMPORTANT]
> 오프라인 사용은 자동 동기화를 뜻하지 않습니다. 휴대전화에서 저장한 생각이 PC에 자동으로 나타나지 않으며, 옮기려면 JSON 내보내기와 가져오기를 사용해야 합니다.

## 손상된 데이터 복구

저장된 데이터가 예상한 형식이 아니면 앱은 추가 손상을 막기 위해 `생각 내용` 입력과 저장한 생각의 작업 버튼을 잠그고 `데이터 점검 필요` 안내를 표시합니다.

가장 먼저 `원본 내려받기`를 눌러 현재 값을 텍스트 파일로 보관하세요. 이 파일은 원본 보존용이며, 손상 정도에 따라 앱으로 다시 가져올 수 있다는 보장은 없습니다.

### 안내 유형별 대응

| 화면 안내 | 뜻 | 권장 대응 |
| --- | --- | --- |
| 일부 저장 항목을 안전하게 읽지 못한 경우 | 유효한 항목과 손상·중복 항목이 섞여 있음 | 원본을 내려받은 뒤 `읽힌 항목으로 복구` |
| 더 새로운 버전에서 만든 보관함이에요 | 현재 앱이 모르는 새 형식의 데이터임 | 가능하면 더 새 버전에서 열기. 계속하려면 원본 백업 후 `현재 형식으로 복구` |
| 저장한 내용을 안전하게 읽지 못한 경우 | 저장 내용 전체를 해석할 수 없음 | 원본을 내려받은 뒤 정상 JSON 백업으로 교체하거나 `비우고 계속` |

### 정상 백업 파일이 있을 때

1. `원본 내려받기`로 현재 손상 데이터를 먼저 보존합니다.
2. 데이터 관리를 열고 `가져오기`에서 정상 JSON 백업을 선택합니다.
3. `파일 내용으로 교체`를 선택합니다.

손상 상태에서는 현재 데이터와 안전하게 합칠 수 없으므로 `기존 보관함에 합치기`가 비활성화됩니다. 백업 파일로 교체한 복구 작업은 되돌릴 수 없습니다.

### 일부 항목을 읽을 수 있을 때

1. 화면에 남아 있는 저장 항목 수와 내용을 확인합니다.
2. `원본 내려받기`로 원본을 보존합니다.
3. `읽힌 항목으로 복구`를 누릅니다.
4. 확인 창의 `현재 형식으로 복구`를 누릅니다.

읽지 못한 항목과 현재 앱이 모르는 새 형식 정보는 제거됩니다. 이 작업도 되돌릴 수 없습니다.

### 아무 항목도 읽을 수 없을 때

정상 백업이 없다면 `원본 내려받기` 후 `비우고 계속`을 선택할 수 있습니다. 확인 창에서 데이터를 지우면 저장한 생각이 없는 상태로 다시 시작합니다.

브라우저가 저장이나 삭제를 차단하면 복구가 완료되지 않으며 원본은 그대로 유지됩니다. 이 경우 브라우저의 사이트 데이터 권한과 저장 공간을 확인한 뒤 다시 시도하세요.

| 서버 전송 | 하지 않음 |

`이 브라우저에 저장`은 정확히는 **현재 사이트 주소와 브라우저 프로필의 저장 공간**에 보관된다는 뜻입니다. 다음 환경은 서로 다른 보관함으로 취급될 수 있습니다.

- Chrome과 Safari처럼 서로 다른 브라우저
- 일반 창과 시크릿·개인정보 보호 창
- 같은 브라우저의 서로 다른 사용자 프로필
- 서로 다른 도메인이나 개발용 주소
- 휴대전화와 PC

다음 작업은 저장한 생각을 잃게 만들 수 있습니다.

- 브라우저의 쿠키·사이트 데이터 삭제
- 브라우저의 자동 삭제 기능이 사이트 저장소를 지움
- 시크릿·개인정보 보호 창 종료
- 브라우저 프로필 삭제 또는 초기화
- 기기 고장이나 저장 장치 손실

중요한 내용은 정기적으로 JSON으로 내보내고, 잠깐 맡김소를 비밀번호·인증 코드·금융 정보·건강 정보 같은 민감한 기록의 유일한 보관 장소로 사용하지 마세요. 같은 기기나 브라우저 프로필에 접근할 수 있는 사람은 개발자 도구나 백업 파일을 통해 내용을 볼 수 있습니다.

## 자주 묻는 질문과 문제 해결

### 고른 시간이 되었는데 알림이 오지 않아요

정상입니다. 이 앱은 운영체제 알림, 푸시 알림, 이메일을 보내지 않습니다. 앱이 열려 있으면 약 30초마다 상태를 확인하고, 닫혀 있었다면 다음 실행 때 `지금 볼 생각`으로 분류합니다.

### 시간이 되었는데 최대 30초 정도 `나중에 볼 생각`에 남아 있어요

열린 화면은 약 30초 간격으로 시간을 확인합니다. 잠시 기다리거나 페이지를 다시 열면 현재 시각을 기준으로 즉시 분류됩니다.

### 저녁 선택지가 `내일 저녁`으로 보여요

정상입니다. 저녁 선택은 오후 8시를 뜻합니다. 현재 시각이 오후 8시 전이면 `오늘 저녁`, 이미 지났으면 `내일 저녁`으로 라벨이 바뀌며 다음 날 오후 8시로 정해집니다.

### `내일 이맘때`와 `내일 아침`은 같은가요?

다릅니다.

- `새 생각`의 `내일 아침`: 다음 달력 날짜 오전 9시
- `지금 볼 생각`에서 `나중에 보기`의 `내일 이맘때`: 버튼을 누른 시각으로부터 24시간 뒤

### 시간을 고쳤는데 예상과 다르게 보여요

앱은 기기의 날짜, 시간, 시간대를 사용합니다. 기기 시계나 시간대가 틀렸다면 바로잡은 뒤 앱을 다시 여세요. `날짜 직접 선택`은 별도의 시간대 선택 없이 현재 기기의 현지 시각으로 해석됩니다.

### 저장한 생각의 내용을 수정하고 싶어요

현재 버전에는 직접 수정 기능이 없습니다. 기존 생각을 삭제한 뒤 원하는 내용과 시간으로 새로 저장해 주세요. 삭제 직후 잘못 지웠다는 것을 알았다면 약 10초 안에 되돌릴 수 있습니다.

### 검색했는데 결과가 없어요

검색은 `생각 내용`만 확인합니다. `나중에 볼 생각` 같은 상태 이름이나 날짜는 검색되지 않습니다. 문장의 일부 단어로 다시 검색하거나 `검색 지우기`를 누르세요.

### 저장이 되지 않을 때

다음을 차례로 확인합니다.

1. 생각 입력란이 비어 있지 않은지 확인합니다.
2. 직접 정한 시간이 현재보다 뒤인지 확인합니다.
3. 화면에 `데이터 점검 필요` 안내가 있다면 먼저 복구 절차를 마칩니다.
4. 상단 배지가 `이 탭에서만 유지`인지 확인합니다.
5. 브라우저가 사이트 저장을 차단했는지, 저장 공간이 부족한지 확인합니다.

`이 탭에서만 유지` 상태에서는 현재 탭 안에서 기능을 계속 사용할 수 있지만, 탭을 닫으면 내용이 사라질 수 있습니다. 상단 배지와 함께 화면 폭에 관계없이 `저장 차단됨` 경고가 계속 표시됩니다. 경고의 `데이터 관리 열기`로 JSON을 내보내고 브라우저 저장 설정을 확인하세요.

### `데이터`에 앱 설치 영역이 보이지 않아요

`앱으로 설치하기` 영역과 `앱 설치` 버튼은 브라우저가 설치 가능하다고 판단한 경우에만 데이터 관리 대화상자에 나타납니다.

- HTTPS 또는 localhost로 열었는지 확인합니다.
- 브라우저 메뉴의 `앱 설치`나 `홈 화면에 추가`를 확인합니다.
- 이미 설치되어 있다면 버튼이 나타나지 않습니다.
- 일부 브라우저는 웹앱 설치를 지원하지 않습니다.

설치는 선택 사항이며, 버튼이 없어도 브라우저에서 앱을 사용할 수 있습니다.

### 오프라인에서 앱이 열리지 않아요

네트워크가 있을 때 앱을 한 번 완전히 열어 오프라인 파일을 저장해야 합니다.

1. 다시 온라인으로 연결합니다.
2. HTTPS 주소 또는 localhost에서 앱을 엽니다.
3. 화면이 모두 열린 뒤 한 번 새로고침합니다.
4. 다시 오프라인으로 전환해 확인합니다.

브라우저의 캐시나 사이트 데이터를 지웠다면 오프라인 화면 파일도 함께 없어질 수 있습니다.

### 다른 기기나 브라우저에 저장한 생각이 없어요

자동 동기화 기능이 없기 때문입니다. 원래 브라우저에서 JSON으로 내보낸 뒤 새 브라우저에서 가져오세요. 두 곳의 내용을 모두 유지하려면 `기존 보관함에 합치기`를 선택합니다.

### 가져오기가 실패해요

잠깐 맡김소에서 내보낸 `.json` 파일인지 확인합니다. 파일이 손상되었거나, 유효한 저장 항목이 없거나, 현재 앱보다 더 새로운 형식이면 안전을 위해 가져오지 않습니다.

파일을 텍스트 편집기로 수정했다면 원본 백업으로 다시 시도하세요. 손상 데이터 복구 때 받은 `.txt` 파일은 원본 보존용이므로 일반 백업 JSON과 같지 않습니다.

### 합치기를 했는데 일부 항목이 늘어나지 않았어요

같은 고유 ID를 가진 항목은 중복으로 추가하지 않습니다. 같은 백업을 여러 번 합쳐도 동일한 생각이 계속 복제되지 않는 것이 정상입니다.

### 전체 삭제를 되돌릴 수 있나요?

앱 안에서는 되돌릴 수 없습니다. 삭제 전에 내보낸 JSON 백업이 있다면 데이터 관리의 `가져오기`와 `파일 내용으로 교체`로 복원할 수 있습니다.

### 완료한 생각은 자동으로 사라지나요?

아니요. `완료한 생각`에 계속 남아 있습니다. 하나씩 `삭제`하거나 데이터 관리에서 보관소 전체를 삭제할 때만 사라집니다.

### 여러 탭에서 동시에 열어도 되나요?

같은 사이트를 같은 브라우저 프로필의 여러 탭에서 열면 한 탭의 저장 변경을 다른 탭이 감지해 화면을 갱신합니다. 다만 동시에 빠르게 변경하면 마지막으로 저장된 상태가 남을 수 있으므로, 중요한 데이터 작업은 한 탭에서 하는 편이 안전합니다.

## 안전하게 사용하는 작은 습관

1. `생각 내용`에는 “무엇을 할지”가 드러나는 짧은 문장을 씁니다.
2. 고른 시간이 와도 못 했다면 죄책감 대신 `나중에 보기`를 사용합니다.
3. 일주일에 한 번 `완료한 생각`을 훑고 필요 없는 기록을 지웁니다.
4. 중요한 내용이 쌓였다면 JSON 백업을 내려받습니다.
5. 민감하거나 장기 보존이 필요한 정보는 별도의 안전한 도구에 보관합니다.

잠깐 맡김소의 목적은 할 일 목록을 하나 더 늘리는 것이 아니라, 지금 해야 하는 일에 돌아갈 수 있도록 생각을 잠시 안전하게 내려놓는 것입니다.
