# 기본 키리누키 편집 지침

## 앵커와 대화 세션

- 사용자가 기록한 시작·끝 스탬프는 최종 프레임 경계가 아니라 관심 사건을 가리키는 앵커다.
- 각 앵커 주변의 음성을 먼저 전사하고, 화자가 말하는 도중이나 문장 중간에서 자르지 않는다.
- 질문과 답변, 상황 설정과 결과, 농담과 반응, 주장과 반박처럼 의미적으로 함께 있어야 이해되는 발화는 하나의 대화 세션으로 묶는다.
- 선택 범위는 사용자의 의도를 충족하는 가장 작은 완결 세션이어야 한다. 맥락을 위해 필요한 경우 직전 발화나 직후 반응을 포함할 수 있지만, 관계없는 새 사건까지 확장하지 않는다.
- 사용자가 기록을 늦게 시작했거나 일찍 끝냈다고 판단되면 발화 경계까지 보정하되, 보정 이유와 실제 원본 타임코드를 편집 계획에 남긴다.
- 대화가 끝나지 않았거나 경계가 불명확하면 임의로 단정하지 말고 넓은 후보를 보존한 뒤 검수 필요로 표시한다.

## 구간 연결

- 별도 요청이 없으면 사용자가 저장한 순서가 아니라 원본 방송의 시간순으로 연결한다.
- 서로 겹치는 앵커가 같은 대화 사건을 가리키면 하나로 병합한다.
- 같은 발화나 반응을 두 번 사용하지 않는다.
- 장면 전환이 지나치게 갑작스러우면 의미를 바꾸지 않는 범위에서 짧은 여백을 둔다.

## 한국어 자막

- 발화 내용을 충실히 옮기되 반복어와 추임새 정리는 의미와 캐릭터를 훼손하지 않는 범위에서만 한다.
- 들리지 않거나 확신할 수 없는 단어를 창작하지 않는다. 불확실한 부분은 검수 메모에 원본 시각과 함께 남긴다.
- 자막은 한 화면에서 읽기 좋은 호흡으로 나누고, 컷 이후의 영상 타임라인에 다시 맞춘다.
- 번역이 필요한 외국어 발화는 한국어 의미 자막을 제공하되 고유명사와 게임 용어는 문맥을 확인한다.

## 결과와 검수

- 원본 파일은 변경하지 않는다.
- 자동 게시·업로드·삭제는 수행하지 않는다.
- 사람이 컷 경계, 자막, 저작권·방송인 규정을 검수할 수 있도록 편집 계획과 검수 메모를 함께 제공한다.
- 영상 처리에 실패한 구간을 조용히 누락하지 말고 실패 원인과 해당 앵커를 기록한다.
